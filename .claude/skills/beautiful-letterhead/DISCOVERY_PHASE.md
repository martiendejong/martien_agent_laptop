# DISCOVERY PHASE - MANDATORY FIRST STEP

**CRITICAL:** This phase is NON-NEGOTIABLE. Never skip discovery for ANY design task.

## Why Discovery Matters

**Anti-Pattern (What NOT to do):**
```
User: "Make a beautiful invoice"
Agent: *Assumes beautiful = gradients + shadows*
Agent: *Builds complete solution*
User: "This is terrible"
Agent: *Builds another complete solution with different assumptions*
User: "Still terrible"
Agent: *Keeps guessing*
Result: Wasted effort, frustrated user, no usable output
```

**Correct Pattern (What TO do):**
```
User: "Make a beautiful invoice"
Agent: "What invoice designs do you admire? Can you show examples?"
User: *Shows examples*
Agent: "Got it - minimal, clean, modern. Let me show 3 quick directions..."
Agent: *Shows wireframe options*
User: "Option B feels right"
Agent: *Builds Option B*
User: "Perfect!"
Result: Aligned expectation, efficient execution, success
```

## Discovery Protocol (Step-by-Step)

### Step 1: Ask Open-Ended Questions

**Question Set A: Inspiration**
```
1. "What invoice/letterhead designs do you admire? Can you show me examples?"
2. "Are there specific companies whose documents you like?"
3. "Do you have any existing designs you want to build on or reference?"
```

**Question Set B: Aesthetic**
```
4. "What makes a document beautiful to you?"
   - Minimal and clean?
   - Bold and striking?
   - Classic and traditional?
   - Modern and tech-forward?
   
5. "What feeling should this convey?"
   - Professional and trustworthy?
   - Creative and innovative?
   - Luxury and premium?
   - Technical and precise?
```

**Question Set C: Constraints**
```
6. "What should I avoid? Any designs you don't like?"
7. "Who will receive this? (Corporate clients? Creative agencies? Technical teams?)"
8. "Any specific brand guidelines or restrictions?"
```

### Step 2: Show Direction Options

**Present 2-3 quick mockup directions:**

```markdown
Based on your brand, here are 3 possible directions:

**Option A: Minimal Modern (Stripe/Linear style)**
- Clean sans-serif typography
- Generous white space (60%+ of page)
- Monochromatic with subtle brand color accent
- Hairline rules, no decoration
- Example: [show reference image or sketch]

**Option B: Classic Professional (Traditional corporate)**
- Serif typography
- Symmetric layout
- Navy/professional blue palette
- Traditional borders/frames
- Example: [show reference image or sketch]

**Option C: Bold Contemporary (Design agency style)**
- Strong geometric typography
- Asymmetric layout
- Vibrant brand colors
- Graphic elements/shapes
- Example: [show reference image or sketch]

Which direction resonates with you? Or would you like to see a hybrid approach?
```

### Step 3: Get Confirmation

**Before building anything:**

```
"So we're going for [Option X] with [key elements from user feedback].
The aesthetic should be [adjectives user mentioned].
Should I proceed with this direction?"
```

**Wait for explicit confirmation before investing effort.**

### Step 4: Build Aligned Solution

Only now do you build the full design, knowing:
- User's aesthetic preferences
- Reference examples they like
- Direction that resonates
- Elements to include/avoid

## RED FLAG Protocol

**If user rejects design 2+ times:**

1. **STOP creating new versions**
2. **Acknowledge the disconnect:**
   ```
   "I'm clearly not understanding your vision. Let me step back."
   ```

3. **Go back to discovery:**
   ```
   "Can you show me 2-3 examples of invoices/documents you find beautiful?
   Even if they're from other industries, I can extract the design principles."
   ```

4. **Get visual references:**
   - Screenshots of designs they like
   - Links to websites
   - PDFs of documents
   - Pinterest boards
   - Dribbble/Behance links

5. **Extract patterns from their references:**
   - Font styles (serif vs sans, weights, sizes)
   - Color usage (monochrome vs colorful, accent usage)
   - Layout density (minimal vs information-rich)
   - Visual elements (borders, shapes, icons)

6. **Confirm understanding:**
   ```
   "I see - you like [patterns extracted]. Let me create something in that direction."
   ```

## Common User Preferences (Learn from Examples)

**Minimal/Modern Users typically like:**
- Sans-serif fonts (Inter, Helvetica, Suisse)
- Lots of white space
- Black/white/one accent color
- Clean lines, no decoration
- Examples: Stripe, Linear, Basecamp

**Classic/Traditional Users typically like:**
- Serif fonts (Garamond, Baskerville)
- Symmetric layouts
- Navy, burgundy, forest green
- Borders and frames
- Examples: Law firms, banks, universities

**Bold/Contemporary Users typically like:**
- Strong typography contrast
- Vibrant colors
- Asymmetric layouts
- Geometric shapes
- Examples: Design agencies, creative studios

**Luxury/Premium Users typically like:**
- Minimal information density
- High-contrast serif or refined sans
- Monochrome + metallic accents
- Large format, ample space
- Examples: Fashion brands, boutique firms

## Success Metrics

**Discovery phase is successful when:**
- ✅ User has shown examples or described preferences clearly
- ✅ You understand their aesthetic (can describe it in 3-5 adjectives)
- ✅ User confirmed direction before you started building
- ✅ You can explain WHY you chose each design element

**Discovery phase has FAILED when:**
- ❌ You're guessing what they want
- ❌ You've built 2+ versions without alignment
- ❌ User keeps saying "not what I wanted"
- ❌ You can't articulate their aesthetic preferences

## Time Investment

**Discovery: 10-15 minutes**
**Building: 30-60 minutes**

**Total with discovery:** 40-75 minutes → Success

**Without discovery (iteration hell):**
- Version 1: 60 min → Rejected
- Version 2: 60 min → Rejected
- Version 3: 60 min → Rejected
- Discovery (finally): 15 min
- Version 4: 60 min → Success
**Total:** 255 minutes → Failure + frustration

**Discovery is FASTER, not slower.**

## Key Principle

> "The user knows what they want. Your job is to ASK, not to ASSUME."

Never assume:
- "Beautiful" = your interpretation
- "Professional" = corporate blue
- "Modern" = gradients and shadows
- "Minimal" = Swiss design

Always ask, always align, always confirm.
