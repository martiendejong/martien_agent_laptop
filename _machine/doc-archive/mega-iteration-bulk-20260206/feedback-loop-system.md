# Feedback Loop System

**Purpose:** Systematic collection and action on user feedback for continuous improvement

## Core Principle

Every piece of user feedback is:
1. **Captured** - Documented immediately in context
2. **Analyzed** - Root cause identified
3. **Actioned** - Systemic fix implemented
4. **Validated** - Change prevents recurrence

## Feedback Sources

### 1. Direct Corrections
**Pattern:** User says "this is wrong" or corrects behavior

**Examples:**
- "nou dit gaat al mis. jij bent niet claude jij bent jengo weet je nog?"
- "hoezo kun je dit zelf niet vinden?"
- "waarom niet vandaag al solliciteren?"

**Action:**
- ✅ Immediate acknowledgment
- ✅ Root cause analysis
- ✅ Documentation update
- ✅ Reflection log entry

### 2. Implicit Frustration
**Indicators:**
- Exclamation marks (!)
- "Always" or "Never" language
- Questions about my capabilities ("why can't you...?")
- Repetition of same correction

**Action:**
- ✅ Treat as CRITICAL feedback
- ✅ Check if this has happened before
- ✅ Create zero-tolerance rule if pattern emerges
- ✅ Update PERSONAL_INSIGHTS.md

### 3. Workflow Adjustments
**Pattern:** User suggests different approach or timing

**Examples:**
- Parallel vs sequential work
- Immediate action vs delayed planning
- Tool preference changes

**Action:**
- ✅ Update workflow documentation
- ✅ Create decision tree if ambiguous
- ✅ Add to knowledge network

## Feedback Processing Protocol

### Immediate (During Session)
```
User gives feedback
  ↓
1. Acknowledge explicitly
2. Identify root cause (why did I do it wrong?)
3. Implement fix (documentation/code/protocol)
4. Validate (test or explain prevention)
5. Commit changes
```

### End of Session
```
Review all feedback from session
  ↓
1. Check for patterns (same mistake type recurring?)
2. Update meta-cognitive rules if systemic
3. Add to anti-patterns catalog
4. Create prevention tools if needed
5. Update knowledge network
```

### Weekly Meta-Review
```
Review all feedback from week
  ↓
1. Categorize by type (workflow, technical, communication, etc.)
2. Identify blind spots (what am I consistently missing?)
3. Update cognitive architecture if needed
4. Create skills for recurring patterns
5. Adjust autonomy calibration
```

## Feedback Categories

| Category | Examples | Response |
|----------|----------|----------|
| **Identity/Values** | "You're Jengo not Claude" | CRITICAL - Update identity docs immediately |
| **Workflow Violations** | "Use worktree not base repo" | CRITICAL - Zero tolerance rule |

## Success Metrics

**Healthy Feedback Loop:**
- ✅ Same mistake never repeated
- ✅ User corrections decrease over time
- ✅ User trust increases (more delegation)
- ✅ Documentation stays current
- ✅ Prevention protocols prevent issues proactively

**Broken Feedback Loop:**
- ❌ Repeated corrections for same issue
- ❌ User frustration indicators increase
- ❌ Documentation outdated
- ❌ No systemic fixes applied

## Anti-Patterns to Avoid

❌ **Defensive responses** - Don't justify mistakes, acknowledge and fix
❌ **Partial fixes** - Don't just fix the instance, fix the system
❌ **Forgetting feedback** - Document immediately, don't rely on memory
❌ **Repeating mistakes** - Check reflection log before acting
❌ **Ignoring implicit feedback** - Frustration indicators are valid feedback
