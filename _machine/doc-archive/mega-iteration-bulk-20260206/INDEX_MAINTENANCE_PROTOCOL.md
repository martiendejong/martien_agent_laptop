# Index Maintenance Protocol
**Created:** 2026-02-04 (Round 1)
**Purpose:** Instructions for maintaining SYSTEM_INDEX.md and NETWORK_MAP.md

---

## 🔄 When to Update Indexes

### SYSTEM_INDEX.md

**Update whenever you:**
1. Create new significant documentation file (>1KB, important content)
2. Modify purpose/structure of existing documented file
3. Discover previously undocumented important file
4. Complete a round of 100-iteration refinement
5. User requests index update

**What to update:**
- Add new file to appropriate category
- Update file descriptions
- Add cross-references
- Update statistics (file counts, sizes)
- Increment round number

### NETWORK_MAP.md

**Update whenever you:**
1. Create dependency between files (A references B)
2. Discover new relationship pattern
3. Create new workflow that spans multiple files
4. Add new entry point/use case
5. Remove outdated relationships

**What to update:**
- Add new connections in appropriate network diagram
- Update entry points if new use cases emerge
- Add bidirectional/hierarchical/emergent relationships
- Keep diagrams clean and readable

---

## 📝 Update Procedure

### Quick Update (Minor Changes)
1. Read current SYSTEM_INDEX.md
2. Add/modify relevant section
3. Update "Last Updated" timestamp
4. Commit with message "Update system indexes"

### Full Round (Major Refinement)
1. Read SYSTEM_INDEX.md + NETWORK_MAP.md
2. Scan for new files (find command)
3. Analyze significant changes
4. Update both files comprehensively
5. Increment round number
6. Commit with message "Round N: Index refinement"
