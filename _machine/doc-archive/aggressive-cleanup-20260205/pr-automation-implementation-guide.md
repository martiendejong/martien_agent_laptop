## ✅ Wat is geïmplementeerd

### 1. Merge Develop First (jouw suggestie!)

**File:** `C:\scripts\tools\worktree-release-all.ps1` (updated)

**Wat het doet:**
```
OUDE workflow:
  Commit → Push → (hopelijk geen conflicts)

NIEUWE workflow:
  Commit → Merge develop → Conflict? STOP! → Pre-flight checks → Push
```

**Code toegevoegd:**
```powershell
# Merge develop into feature branch BEFORE push
git fetch origin develop
git merge origin/develop --no-edit

if (conflicts detected) {
    STOP - show conflict files
    User must: resolve → git add → git merge --continue → re-run script
}
```

**Voordelen:**
- ✅ PR toont **merged state** (niet isolated changes)
- ✅ Conflicts gevonden **lokaal** (niet in GitHub)
- ✅ CI test **actual merge result**
- ✅ Geen "worked on my machine" failures

### 2. Pre-Flight Validation

**File:** `C:\scripts\tools\pr-preflight.ps1` (NEW)

**Checks uitgevoerd:**

| Check | Wat het doet | Auto-fix? |
|-------|--------------|-----------|
| **1. Branch sync** | Verify merged with develop | ❌ |
| **2. Backend build** | `dotnet build` | ❌ |
| **3. Frontend build** | `npm run build` | ❌ |
| **4. EF migrations** | Check pending model changes | ✅ (creates migration) |
| **5. Unit tests** | Run fast tests only | ❌ |
| **6. Static analysis** | Integrates `auto-code-review.ps1` | ❌ |
| **7. Code quality** | TODO/debug statement count | ❌ |

**Exit codes:**
- `0` = All checks passed → Safe to create PR
- `1` = Failures detected → Fix before PR creation
- `2` = Warnings only → Can proceed with caution

**Integration:**
`worktree-release-all.ps1` automatically calls `pr-preflight.ps1` after merging develop.

### 3. Automated Workflow

**File:** `C:\scripts\tools\worktree-release-all.ps1` (updated)

**Nieuwe workflow:**

```
Developer klaar met feature
    ↓
Run: worktree-release-all.ps1 -AutoCommit
    ↓
1. Commit uncommitted changes
    ↓
2. Merge develop into feature branch
    ↓ (if conflicts → STOP, user fixes, re-runs)
    ↓
3. Run pr-preflight.ps1
    ↓ (if fails → STOP, user fixes, re-runs)
    ↓
4. Push validated + merged code
    ↓
5. Switch to base branch
    ↓
6. Mark worktree FREE
    ↓
✅ Ready for PR creation (manual: gh pr create)
```

## 💡 Tips & Best Practices

### When to use pr-preflight.ps1 manually

**Use standalone for quick validation:**
```powershell
# Before committing (catch issues early):
.\pr-preflight.ps1 -SkipTests  # Fast check (30s)

# Before pushing (full validation):
.\pr-preflight.ps1 -Verbose    # See all details

# After fixing issues:
.\pr-preflight.ps1 -AutoFix    # Auto-create migration if needed
```

### Integration with existing workflow

**Option 1: Use worktree-release-all.ps1 (recommended)**
```powershell
# One command does everything:
.\worktree-release-all.ps1 -AutoCommit
```

**Option 2: Manual steps**
```powershell
# For more control:
git add -A
git commit -m "..."
git fetch origin develop && git merge origin/develop --no-edit
.\pr-preflight.ps1
git push
gh pr create
```

### Troubleshooting

**Pre-flight fails with "Cannot find module":**
```powershell
# Frontend build needs dependencies:
cd <frontend-dir>
npm install
```

**EF migration check fails:**
```powershell
# Ensure EF tools installed:
dotnet tool install --global dotnet-ef

# Or use auto-fix:
.\pr-preflight.ps1 -AutoFix
```

**Script can't find repo:**
```powershell
# Explicit repo parameter:
.\pr-preflight.ps1 -Repo "client-manager" -Branch "your-branch"
```
