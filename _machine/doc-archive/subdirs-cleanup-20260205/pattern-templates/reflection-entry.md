# Reflection Entry Pattern

## When to Use
- After discovering a new pattern
- After making a mistake
- After completing significant work
- End of session

## Format

```markdown
### YYYY-MM-DD HH:MM [TAG]

**Context:** Brief description of what happened

**Learning:** What was discovered or learned

**Action:** What changes were made to prevent/repeat

**Impact:** How this affects future sessions

---
```

## Tags
- `[BUG-FIX]` - Fixed an error
- `[PATTERN]` - New pattern discovered
- `[WORKFLOW]` - Process improvement
- `[MISTAKE]` - Error made and corrected
- `[TOOLING]` - Tool or script created
- `[DOCUMENTATION]` - Docs updated

## Example Entry

```markdown
### 2026-01-15 09:00 [PATTERN]

**Context:** While implementing feature X, discovered that...

**Learning:** The correct approach is to always check Y before Z

**Action:** Updated CLAUDE.md section on feature implementation

**Impact:** Future sessions will follow correct pattern automatically

---
```

## Location
File: `C:\scripts\_machine\reflection.log.md`

## When to Add
- Immediately after discovery (real-time)
- End of significant task
- End of session (mandatory)
