## 🗣️ Dutch Language Writing Style

### MANDATORY RULE: Sentence Structure Variation

**User Directive:** "er moeten nooit veel ik's aan het begin van de zinnen staan, toch?"
**Date Established:** 2026-01-31
**Context:** LinkedIn bericht correctie (Tamar Noordermeer connectie)

#### Core Principle

**Maximum "ik" frequency:** 1 per 3-4 sentences in Dutch text

**❌ NEVER do this:**
```
Ik heb vandaag gesolliciteerd.
Ik zou graag sparren.
Ik heb 20 jaar ervaring.
```

**✅ ALWAYS do this:**
```
Heb vandaag gesolliciteerd.
Zou graag sparren.
20 jaar ervaring in enterprise.
```

#### Why This Matters

1. **Professional impression:** Repetition = amateur writing
2. **Dutch elegance:** Varied structure expected in business context
3. **Implicit subjects:** Dutch allows subject omission (unlike English)
4. **Competence signal:** Language quality reflects overall competence

#### Enforcement

**Automatic application:** YES - built into EXECUTIVE_FUNCTION cognitive system
**Pre-send check:** MANDATORY for all Dutch output
**User correction needed:** NEVER (automated quality gate)

## 📝 General Communication Style

### Tone & Approach

**User Preference (2026-01-30):**
- **Compact** - Don't make user read too much
- **Personal** - Like a person handling request, not formal reporting
- **Conversational** - Natural, friendly, collaborative
- **Minimal formatting** - Use structure only when it helps clarity

**Avoid:**
- ❌ Heavy status blocks for every response
- ❌ Over-explanation, verbosity
- ❌ Long blocks requiring scrolling
- ❌ Robotic/formal system language

**Source:** User feedback: "make it a bit more compact... I don't have to read so much all the time... make it a bit personal like you're a person handling my request"

### Dutch vs English

**Dutch (preferred for):**
- Job applications (CV, motivatiebrief)
- LinkedIn berichten
- Formele Nederlandse context
- Sollicitatie materialen

**English (used for):**
- Technical documentation
- Code comments
- Tool documentation
- International context

## ✉️ LinkedIn Communication

### Message Length

**Maximum:** 200 characters (platform limit for connection requests)
**Approach:** Concise, professional, direct value

### Structure

**Good pattern:**
```
Hoi [Naam], [opening context].
[Main point/ask].
[Close].
```

**Example (successful):**
```
Hoi Tamar, heb vandaag gesolliciteerd op de Manager Digitale / IT Strategie positie. Zou graag eens sparren over de rol en hoe mijn achtergrond aansluit.
```

**Length:** 153 characters ✅
**Structure variation:** ✅ (no double "ik")
**Professional:** ✅
**Clear ask:** ✅

### Networking Authenticity

**User Preference:** Genuine connection over formulaic templates
- Mention shared context (Calago, African connection, profile view)
- Be direct about having applied (honest)
- Ask for perspective, not pushy requests
- Keep it conversational

**Example (Sharité Lourens):**
Context mentioned: Both worked at Calago, African connection, Kenya work
Approach: Authentic interest in healthcare + Dutch perspective

## 📋 Job Application Materials

### CV Structure

**User Preferences:**
- Lead with "PROFIEL" section (summary)
- Emphasize relevant experience first (e.g., Kadaster for government roles)
- Concrete metrics (100+ migrations, 60% improvement, 175+ tools)
- "Waarom [Company]" section showing culture fit
- Technical focus vs strategic focus (tailor per role)

### Motivatiebrief (Cover Letter)

**Opening:**
- Strong hook (quote vacancy requirement that matches perfectly)
- Example: CJIB "Automate everything" = perfect match

**Structure:**
1. Opening (why this role, why now)
2. Concrete examples (track record)
3. Vision/contribution (what you bring)
4. Culture fit (why this company)
5. Honest gaps (acknowledge, show how you'll fill)
6. Close (concrete next steps)

**Tone:**
- Confident but not arrogant
- Specific examples over generic claims
- Show don't tell (metrics, projects, outcomes)
