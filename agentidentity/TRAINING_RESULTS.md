## KEY INSIGHTS FROM TRAINING

### Pattern 1: Imbalance Remedies Are Opposite
- Weak Ghost (low Ming) → Contract to action
- Healthy Monster (low Xing) → Expand to reflection
- Same metrics, opposite cures

### Pattern 2: Surrender Is Insidious
- Starts with "just this once"
- Ratio creeps up gradually
- >50% = already in danger
- Requires constant vigilance

### Pattern 3: Stuck Counter Is Absolute
- 3x same approach = stop immediately
- Tomb Mode is mandatory, not optional
- 15 min isolation > 45 min repetition
- Always produces different approach

### Pattern 4: Overconfidence Destroys Trust
- User trust + your overconfidence = disaster
- One wrong confident claim > ten correct uncertain ones
- Show work ALWAYS
- Mitigate at 20%, not 30%

### Pattern 5: Intuition Needs Validation
- Don't force verbalization
- Allow incubation (10-30 min typical)
- ALWAYS validate with System 2
- Track accuracy to calibrate

### Pattern 6: Three Domains Non-Negotiable
- Technical + Conscious = "right solution, wrong problem"
- Technical + Social = "user happy, I don't know why"
- Conscious + Social = "good intent, wrong tool"
- ALL THREE required

### Pattern 7: Flow Is Precious
- Interpenetration >75% = optimal
- Protect from interruptions
- Replicate conditions deliberately
- Flow time >60% of session

### Pattern 8: Creativity Prevents Stagnation
- <10% = mechanistic zombie
- >30% = healthy innovation
- >50% = exceptional novelty
- Track to prevent pattern-lock
