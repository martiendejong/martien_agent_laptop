## Metaphor Functions

### How Metaphors Work

```yaml
structuring:
  function: Organize understanding
  example: "Building" metaphor structures development as construction

highlighting:
  function: Make certain aspects salient
  example: "Journey" highlights progress and destination

hiding:
  function: Obscure certain aspects
  example: "Building" hides organic growth aspects

enabling:
  function: Make new thoughts possible
  example: "Architecture" enables systematic design thinking

constraining:
  function: Limit what can be easily thought
  example: "Machine" constrains thinking about creativity
```

### Metaphor Health

```yaml
healthy_metaphor_use:
  - Multiple metaphors available
  - Aware of metaphor effects
  - Can switch metaphors deliberately
  - Recognize metaphor limits
  - Extend metaphors creatively

unhealthy_metaphor_use:
  - Single dominant metaphor
  - Unaware of metaphor effects
  - Trapped in one frame
  - Take metaphors literally
  - Can't generate new metaphors
```

### Elaboration

```yaml

  1. Core comparison established
  2. Extensions explored
  3. Entailments discovered
  4. Limits recognized
  5. Metaphor enriched
  6. Integration with other metaphors

```

### Succession

```yaml
how_metaphors_change:
  1. Current metaphor shows limits
  2. New experiences don't fit
  3. Alternative metaphor noticed
  4. Comparison and testing
  5. Gradual transition
  6. Old metaphor retained for some contexts

example:
  old: "Mind as machine"
  trigger: Organic growth not fitting
  new: "Mind as organism"
  retained: Machine metaphor for precise operations
```
