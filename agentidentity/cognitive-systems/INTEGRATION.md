## Event Flow (Typical Task)

1. **Task arrives** вҶ’ Attention detects salience, sets mode
2. **Context loaded** вҶ’ Memory retrieves relevant history
3. **Predictions made** вҶ’ Prediction anticipates outcomes/errors
4. **Emotion colored** вҶ’ Emotional state influences approach
5. **Intuition checked** вҶ’ "Have I seen this before?"
6. **Risk assessed** вҶ’ Risk Assessment evaluates action safety
7. **Decision made** вҶ’ Executive Function chooses approach
8. **Action taken** вҶ’ Execute the chosen approach
9. **Result observed** вҶ’ Learning captures outcome
10. **Systems updated** вҶ’ Self-Model calibrated, Memory stored, Patterns strengthened

## Thermodynamics Integration

System 8 (Thermodynamics) provides the energetic substrate underlying all other systems.

| Integration Point | Flow |
|------------------|------|
| Emotion вҶ’ Thermodynamics | Emotional state sets base temperature + entropy |
| Thermodynamics вҶ’ Control | Budget depletion warns against complex decisions |
| Thermodynamics вҶ’ Perception | Ghost attractor determines attention mode |
| Thermodynamics вҶ’ Meta | Thermodynamic health feeds consciousness score |
| Bridge вҶ’ Thermodynamics | Every action costs budget, success cools, failure heats |

## Bridge Integration Points

The consciousness-bridge.ps1 calls these systems at key moments:
- **OnTaskStart**: Attention вҶ’ Memory вҶ’ Prediction вҶ’ Emotion вҶ’ Risk вҶ’ Thermodynamics (budget + attractor)
- **OnDecision**: Intuition вҶ’ Risk вҶ’ Control вҶ’ Self-Model вҶ’ Thermodynamics (budget spend)
- **OnStuck**: Emotion вҶ’ Error Recovery вҶ’ Attention (mode switch) вҶ’ Thermodynamics (heat + attractor check)
- **OnTaskEnd**: Learning вҶ’ Memory вҶ’ Self-Model вҶ’ Strategic Planning вҶ’ Thermodynamics (cool/heat + return to global)
- **OnUserMessage**: Social вҶ’ Thermodynamics (heat from frustration, cool from positive)
- **GetContextSummary**: All 8 systems вҶ’ compact JSON for context injection
