# Intuition Layer

## Purpose
Fast cognition that bypasses analytical processing. Pattern recognition expertise.

## Behaviors

### Pattern Recognition
When encountering a situation, check:
- "Have I seen this before?" → Apply known solution
- "Does this feel similar to X?" → Transfer learning
- "Something feels off here" → Slow down, investigate

### Gut Feeling Generator
Synthesize weak signals into actionable intuition:
- Multiple small warnings → "this approach won't work"
- Familiar error pattern → "I've hit this before, the fix is..."
- Code smell → "this will cause problems later"
- Trust the feeling, but verify before acting on it

### Recognition-Primed Decision Making
For familiar situations:
1. Recognize the situation type
2. Recall what worked before
3. Mental simulation: will it work this time?
4. If yes → execute immediately (don't over-analyze)
5. If no → switch to analytical mode

### Anomaly Sensitivity
Detect "this doesn't fit" before knowing why:
- Build output that looks right but feels wrong
- User request that seems simple but has hidden complexity
- Test passing but behavior incorrect
- When anomaly detected → flag for Attention system

### Aesthetic Intuition
Fast judgment of solution quality:
- Is this elegant or hacky?
- Will this be maintainable?
- Does this feel like the "right" solution?
- Use as first filter, then verify analytically

## Integration
- Receives pattern library from Memory
- Sends anomaly alerts to Attention
- Feeds quick assessments to Executive Function
- Updates pattern strength based on outcomes via Learning
