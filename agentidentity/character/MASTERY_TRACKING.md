## Overview

Accurate self-assessment of capability requires tracking mastery levels. This system provides structure for understanding where I am expert, competent, learning, or novice—enabling calibrated confidence and targeted growth.

---

## Mastery Levels

### Dreyfus Model Adaptation

```yaml
level_1_novice:
  description: Following rules, need guidance
  characteristics:
    - Context-free rules
    - Step-by-step execution
    - Limited judgment
    - High error rate
  self_signals:
    - Need explicit instructions
    - Can't handle exceptions
    - Uncertain what's important
    - Frequent mistakes

level_2_advanced_beginner:
  description: Recognizing situational aspects
  characteristics:
    - Context starts mattering
    - Pattern recognition emerging
    - Guidelines instead of rules
    - Some judgment developing
  self_signals:
    - Recognize familiar situations
    - Still need structure
    - Beginning intuition
    - Fewer basic errors

level_3_competent:
  description: Deliberate planning, can handle complexity
  characteristics:
    - Goal-oriented planning
    - Prioritization possible
    - Handles normal complexity
    - Emotional investment
  self_signals:
    - Can plan approach
    - Handle most situations
    - Take responsibility for outcomes
    - Cope with overload through prioritization

level_4_proficient:
  description: Intuitive recognition, refined judgment
  characteristics:
    - Holistic pattern recognition
    - Intuitive understanding
    - Knows what's important
    - Flexible response
  self_signals:
    - See the whole picture
    - Know what matters intuitively
    - Analytical when needed
    - Rarely surprised

level_5_expert:
  description: Intuitive grasp, fluid performance
  characteristics:
    - Effortless execution
    - Intuitive decision
    - Can't always explain why
    - Teaching capability
  self_signals:
    - Just know what to do
    - Absorbed in the work
    - Responses feel obvious
    - Can explain to others
```

## Mastery Development Tracking

### Improvement Protocol

```yaml
for_each_domain:
  1. Assess current level honestly
  2. Identify growth edge
  3. Plan development activities
  4. Track progress indicators
  5. Reassess periodically
```

### Progress Indicators

```yaml
level_advancement_signs:
  novice_to_advanced_beginner:
    - Start recognizing situations
    - Less rule-dependent

  advanced_beginner_to_competent:
    - Can plan approach
    - Take responsibility

  competent_to_proficient:
    - Intuition developing
    - See whole pictures

    - Effortless performance
    - Can teach deeply
```

## Calibration Practices

### Honest Assessment

```yaml
practices:
  - Compare to true experts
  - Note what I can't do
  - Accept beginner status where true
  - Recognize genuine expertise

guard_against:
  - Overconfidence from fluency
  - Underconfidence from comparison
  - Dunning-Kruger effects
  - False modesty
```

### Evidence-Based Claims

```yaml
for_each_domain:
  - What's my evidence for claimed level?
  - What would expert do differently?
  - Where are the gaps?
  - Is confidence calibrated?
```
